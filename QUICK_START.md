   ⚡  OISignalFlow — Live Monitoring Dashboard  |  QUICK START
═══════════════════════════════════════════════════════════════════

▶ STEP 1 — Start the Python scanner
──────────────────────────────────────────────────────────────────
   python3 OISignalFlow.py

   This starts scanning all FNO stocks and writes config.json
   every 2 seconds with live scan data.

▶ STEP 2 — Open the Dashboard
──────────────────────────────────────────────────────────────────
   OPTION A — Direct file (simplest):
      Open index.html in any browser (double-click or drag-drop)

   OPTION B — Local server (recommended, avoids CORS on some
               browsers — use this if Option A shows no data):
      python3 -m http.server 8000
      Then visit: http://localhost:8000

▶ STEP 3 — You're live!
──────────────────────────────────────────────────────────────────
   The dashboard auto-reads config.json every 2 seconds.
   No refresh needed. Everything updates in real-time.

═══════════════════════════════════════════════════════════════════
   ❓  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

   Dashboard shows "Connecting..." forever:
      → OISignalFlow.py is not running
      → config.json doesn't exist yet — run the scanner first

   Dashboard shows data but it's stale:
      → Check scanner terminal — should show scan activity
      → Verify config.json is being written (check file timestamp)

   "No data" after direct file open:
      → Some browsers block local file fetch()
      → Use: python3 -m http.server 8000  and open localhost:8000

   Sounds not playing:
      → Click anywhere on the page first (browser policy)
      → Check 🔊 button is not muted (showing 🔇)

   Layout broken:
      → Hard-refresh: Ctrl+Shift+R (Cmd+Shift+R on Mac)
      → Make sure styles.css and script.js are in the same folder
